## 3. Extract forge-music to vault root when declared

`src/welcome.ts` — extend `runFirstRunCheck` with conditional
forge-music extraction. The extraction is **gated on the user's
forge.toml declaring "music"** so non-music vaults don't get
unwanted files.

```typescript
async function ensureBundledForgeMusic(app: App): Promise<void> {
  const adapter = app.vault.adapter;

  // Only extract if the vault declares "music" as a domain.
  const tomlPath = 'forge.toml';
  if (!(await adapter.exists(tomlPath))) return;
  const tomlBody = await adapter.read(tomlPath);
  if (!/domains\s*=\s*\[[^\]]*"music"[^\]]*\]/.test(tomlBody)) return;

  // Same skip-if-exists semantics as forge-moda extraction.
  const targetDir = 'forge-music';
  if (await adapter.exists(targetDir)) return;

  const sourceDir = '.obsidian/plugins/forge-client-obsidian/assets/vaults/forge-music';
  if (!(await adapter.exists(sourceDir))) {
    console.warn('Forge: bundled forge-music missing from plugin assets; skipping extraction');
    return;
  }

  await copyDirRecursive(adapter, sourceDir, targetDir);
  console.log('Forge: extracted bundled forge-music into vault');
}
```

Wire after the existing `ensureBundledForgeModa` call. Reuse
`copyDirRecursive` from `src/copy-dir-core.ts` (already exists from
v0.2.13).

The TOML-domain regex is the load-bearing condition — it must
match forge.toml's `domains = ["music"]` AND `domains = ["music", "other"]`,
but NOT a commented-out `# domains = ["music"]`. The current regex
catches the active-declaration case; if a commented variant becomes
an issue, tighten later.
