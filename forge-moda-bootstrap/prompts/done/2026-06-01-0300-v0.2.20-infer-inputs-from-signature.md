# v0.2.20 — infer inputs from Python signature (option b)

## Why this prompt exists

User authored `Greet.md` with English describing `name`, LLM
generated `def compute(context, name): print("hello " + name)`. The
frontmatter still says `inputs:` (empty). Forge-click reads
frontmatter inputs (empty list), skips the modal entirely, calls
`computeSnippetWithArgs(..., kwargs={})`. The engine's
`exec_python` calls `fn(context, *[], **{})` against a signature
that needs `name` → `TypeError: compute() missing 1 required
positional argument: 'name'`.

Frontmatter declarations drift from the LLM-generated Python
signature. **Engine-side fix (option b)**: parse the Python
signature at runtime, derive the set of inputs from the live
signature, request them via the modal. The Python signature
becomes the source of truth for what `compute` actually needs.
Frontmatter declarations are layered ON TOP (declared-first for
display ordering), but missing-from-frontmatter signature params
are still surfaced to the user.

This makes the snippet contract self-consistent — there's no way
for the LLM to write Python the modal won't accommodate. The user
never has to maintain frontmatter + Python signature in sync.

Out of scope: changing the `/generate` request shape, the LLM's
contract for declaring inputs, or any α-side behavior.

## 2. Fix shape

### 2.1 Python helper

`src/pyodide-host.ts` inlined Python block, alongside
`_forge_get_generate_inventory`, `_forge_sync_user_file`,
`_forge_preflight_then_inventory`:

```python
def _forge_get_input_names(snippet_id: str) -> list:
    """v0.2.20: derive the inputs to request from the user by
    parsing the Python facet's compute() signature, then unioning
    with the frontmatter-declared inputs list.

    The Python signature is the source of truth for what compute()
    actually needs — the LLM may write a signature with params not
    declared in frontmatter, and the modal must still surface those
    so the user can supply values.

    Ordering: declared inputs first (preserves user-intended UI
    order), then any signature-only params appended at the end.

    Defensive fallbacks:
    - malformed Python (SyntaxError): return declared.
    - missing Python facet: return declared.
    - missing compute() function: return declared.
    """
    snip = _forge_resolver.resolve(snippet_id)
    meta = snip.get("meta") or {}
    declared = [str(i) for i in (meta.get("inputs") or [])]

    body = snip.get("body") or ""
    try:
        code = extract_python(body)
    except Exception:
        return declared

    if not code or not code.strip():
        return declared

    import ast
    try:
        tree = ast.parse(code)
    except SyntaxError:
        return declared

    sig_args = None
    for node in ast.walk(tree):
        if isinstance(node, ast.FunctionDef) and node.name == "compute":
            # Collect positional-or-keyword + keyword-only params.
            sig_args = [a.arg for a in node.args.args] + [a.arg for a in node.args.kwonlyargs]
            break
    if sig_args is None:
        return declared

    # Filter out 'context' wherever it appears (typically first).
    sig_args = [a for a in sig_args if a != "context"]

    # Union: declared first, then sig-only.
    out = list(declared)
    for a in sig_args:
        if a not in out:
            out.append(a)
    return out
```

### 2.2 TS surface

`PyodideHostInstance` interface + impl. Mirror the
`getGenerateInventory` / `preflightThenInventory` pattern:

```typescript
/** v0.2.20: derive the inputs to request from the user by parsing
 *  the Python facet's compute() signature, unioned with the
 *  frontmatter-declared inputs. Used by the Forge-click modal to
 *  decide which inputs to ask for. */
getInputNames(snippetId: string): Promise<string[]>;

async getInputNames(snippetId: string): Promise<string[]> {
  this.pyodide.globals.set('_forge_input_names_snippet_id', snippetId);
  const proxy = this.pyodide.runPython(
    `_forge_get_input_names(_forge_input_names_snippet_id)`,
  );
  return this._unwrap(proxy) as string[];
}
```

### 2.3 JS-side: replace frontmatter-only inputs with signature-augmented

`src/main.ts:1498-1518` currently does:

```typescript
const snippetId = view.file.basename;
const vaultPath = (this.app.vault.adapter as any).basePath as string;
const frontmatter = this.app.metadataCache.getFileCache(view.file)?.frontmatter;

const inputs: string[] = frontmatter
  ? (frontmatter.inputs ?? [])
  : (this.lookupInventoryInputs(snippetId) ?? []);

if (inputs.length > 0) {
  // ... open modal
} else {
  // ... compute with empty kwargs
}
```

Replace the `inputs` derivation with a call to the new helper.
Keep the existing frontmatter/lookupInventoryInputs path as a
synchronous fallback for the very-first call before Pyodide finishes
loading (a real edge case — the modal might open before Pyodide
boot completes; fallback uses the stale-but-cheap frontmatter read):

```typescript
const snippetId = view.file.basename;
const vaultPath = (this.app.vault.adapter as any).basePath as string;
const frontmatter = this.app.metadataCache.getFileCache(view.file)?.frontmatter;

// v0.2.20: Python signature is source of truth for which params
// compute() needs. Get the augmented list from the engine; fall
// back to frontmatter-only if Pyodide isn't ready yet.
let inputs: string[];
try {
  const hostManager = getPyodideHost();
  if (hostManager) {
    const host = await hostManager.getInstance();
    inputs = await host.getInputNames(snippetId);
  } else {
    throw new Error('Pyodide host not wired');
  }
} catch (e) {
  console.warn(`Forge: signature-inferred inputs unavailable for '${snippetId}', falling back to frontmatter`, e);
  inputs = frontmatter
    ? (frontmatter.inputs ?? [])
    : (this.lookupInventoryInputs(snippetId) ?? []);
}

if (inputs.length > 0) {
  // ... existing modal-open path, unchanged
} else {
  // ... existing empty-kwargs compute path, unchanged
}
```

The try/catch is non-fatal — if the engine query fails, behavior
degrades to frontmatter-only (matching v0.2.19 and earlier).
