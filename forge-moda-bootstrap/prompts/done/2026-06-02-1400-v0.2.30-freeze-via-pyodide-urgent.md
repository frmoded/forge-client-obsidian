Replace with Pyodide-first, HTTP fallback only if `_pyodideHost`
isn't wired (defensive for dev mode):

```typescript
export async function freezeEdge(
  serverUrl: string,
  vaultPath: string,
  caller: string,
  callee: string,
  state: 'frozen' | 'live',
): Promise<{ status: number; json: any }> {
  // v0.2.30: route through Pyodide. The engine's snapshots.py has
  // the right function; HTTP was just the legacy delivery path.
  // Closed beta has no uvicorn, so the HTTP fallback effectively
  // dead-code's for production. Kept for dev-mode where uvicorn IS
  // running.
  if (_pyodideHost) {
    try {
      const host = await _pyodideHost.getInstance();
      await host.setEdgeState(caller, callee, state);
      return {
        status: 200,
        json: { status: 'ok', caller, callee, state },
      };
    } catch (e) {
      const msg = e instanceof Error ? e.message : String(e);
      console.error('Forge freeze failed:', e);
      return { status: 500, json: { detail: msg } };
    }
  }

  // HTTP fallback — dev mode only.
  const res = await requestUrl({
    url: `${serverUrl}/freeze`,
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({ vault_path: vaultPath, caller, callee, state }),
    throw: false,
  });
  return { status: res.status, json: res.json };
}
```

The status-200 envelope mirrors what the engine's HTTP `/freeze`
endpoint returns; main.ts callers continue branching on `status`
unchanged.

Remove the v0.2.6 NOTE comment at server.ts:112-115 since it's no
longer accurate — `/freeze` now works in closed beta.
