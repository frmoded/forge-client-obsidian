# Doc-only — call out the `# Dependencies` section gotcha for hand-authored snippets

## Scope

Add a short authoring-guidance note to the snippet-authoring documentation (or wherever the closest analogue lives in `forge-client-obsidian` / `forge` repos) explaining that snippets which never go through `/generate` won't auto-emit a `# Dependencies` block in their body, and that the wikilink-context-menu freeze affordance (v0.2.41+) needs that block to have wikilinks to click on. Two paragraphs at most.

No code changes. No release cut. Doc-only commit.

## Why

v0.2.41 wikilink-context-menu freeze surfaced this: `smoke-v0.2.13/hello_random.md` was hand-authored without `# Dependencies`, so right-clicking inside the body found no wikilinks → menu suppressed correctly per the helper's logic, but the user expected the affordance to work. The fix path is documentation, not code: per constitution B7, the `# Dependencies` block is engine-emitted at `/generate` time. Hand-authored snippets that didn't go through `/generate` don't get one. Authoring guidance should call this out so users know what to expect (and that they can hand-author the section if they want the affordance now).

## Implementation notes

### Proposed text

Two paragraphs. Concrete, not theoretical. Suggested wording (adapt to whatever doc's voice):

> **`# Dependencies` is engine-emitted, not author-emitted.** When you write a snippet by hand (creating `.md` files in your vault without going through `/generate`), the snippet won't have a `# Dependencies` block at the bottom of its body. That's expected — per constitution B7, the block is generated by static analysis on the Python facet at `/generate` time, not authored manually.
>
> **What you lose without it:** the wikilink right-click freeze affordance (v0.2.41+) needs wikilinks in the snippet body to operate on. If your hand-authored snippet's body has no `[[snippet_name]]` references — only backticked code like `context.compute('snippet_name', ...)` — the right-click menu has nothing to target, so the freeze/unfreeze items won't appear. **Workaround**: append a `# Dependencies\n\n[[callee_one]] [[callee_two]]\n` block manually. The block doesn't change snippet behavior; it only enables the right-click affordance. The Cmd+P → "Freeze edge" command-palette path works regardless of whether `# Dependencies` is present.

Adjust phrasing to match the destination doc's voice. The two load-bearing things to convey: (1) the gap is real and known, (2) the manual workaround restores the affordance.
