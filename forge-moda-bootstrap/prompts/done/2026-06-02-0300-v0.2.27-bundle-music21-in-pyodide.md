# v0.2.27 — bundle music21 wheel into Pyodide so the music domain actually works

## Why this prompt exists

v0.2.26 resolved the snippet-ID and caller-scoped bare-reference bugs.
The next failure the user hit on forge-clicking `forge-music/blues/song.md`:

```
NameError: name 'key' is not defined
File "<snippet>", line 8, in compute   # form.md: k = key.Key(tonic, mode)
```

Triage:

`forge/core/executor.py:14-30`:

```python
try:
  import music21
  _MUSIC21_NAMES = {
    "music21": music21, "stream": music21.stream, "note": music21.note,
    "chord": music21.chord, "meter": music21.meter, "key": music21.key,
    "tempo": music21.tempo, "pitch": music21.pitch,
    "duration": music21.duration, "instrument": music21.instrument,
    "harmony": music21.harmony, "roman": music21.roman,
  }
except ImportError:
  _MUSIC21_NAMES = {}
```

`forge-client-obsidian/src/pyodide-host.ts:228`:

```typescript
await pyodide.loadPackage(["numpy", "pyyaml", "micropip"]);
```

Pyodide loads only numpy + pyyaml + micropip. `import music21` fails
silently in Pyodide, the `_MUSIC21_NAMES` dict is empty, and the
"music" domain bundle injected into snippet local_ns has only
`forge.music.lib.{sequence, bar, voices, repeat, pentatonic}` —
which themselves call into music21 and would crash if reached, but
the failure happens earlier at `key.Key(...)` in form.md.

CC's own CPython smoke of the same song body returned a clean
`music21.stream.Score` because the venv has music21 installed; the
bug is Pyodide-specific.

**Closed-beta constraint** (INSTALL.md "Everything Pyodide needs is
bundled locally — there's no network fetch"): we can NOT
micropip-fetch music21 from PyPI at runtime. The wheel has to be
vendored.

**music21 8.3.0 wheel facts** (pip-confirmed at prompt-authoring time):
- `music21-8.3.0-py3-none-any.whl` — 22.8 MB, pure-Python, Python 3
  agnostic (works on Pyodide's Python 3.13).
- Declared deps: `chardet, joblib, jsonpickle, matplotlib,
  more-itertools, numpy, requests, webcolors`.
- numpy is already in the bundle. matplotlib + requests are
  heavyweight optionals the snippets we care about don't need
  (matplotlib = plotting; requests = network fetches). joblib =
  parallel processing, probably skippable.

**Size impact**: current release zip is 11.22 MB; adding music21 alone
takes it to ~34 MB. Closed-beta priority outweighs zip size — flag
in feedback but don't gate the release on it.

### 1.3 — Apply the fix

#### Vendor the wheels

Wheels live in `assets/pyodide/` alongside the existing
numpy/pyyaml/micropip wheels. Use the local pip-download path:

```bash
# CC: run this from the plugin repo. The Python version doesn't
# matter — music21 + its pure-Python deps are version-agnostic.
mkdir -p /tmp/m21-wheels
cd /tmp/m21-wheels
~/projects/forge/.venv/bin/python -m pip download \
  --no-deps music21==8.3.0 chardet jsonpickle more-itertools webcolors
# Inspect what landed.
ls -lh /tmp/m21-wheels/
# Copy py3-none-any wheels into the plugin's assets/pyodide/.
cp /tmp/m21-wheels/*py3-none-any.whl \
   ~/projects/forge-client-obsidian/assets/pyodide/
```

**Skip on purpose**: matplotlib, requests, joblib. If the boot
fails with ModuleNotFoundError for one of these, vendor the
missing wheel too — but try the minimum set first.

**Pin versions**: capture exact wheel filenames in `setup-assets.mjs`
and `build-release-zip.mjs` `REQUIRED_FILES` so a future
`npm run setup-assets` doesn't silently pull a newer music21 that
breaks the snippet bodies.

#### Wire micropip install into the host boot

`src/pyodide-host.ts:228` — extend the boot sequence:

```typescript
// v0.2.27: vendor + install music21 + minimum deps so the music
// domain (gated by domains=["music"] in vault forge.toml) actually
// has key, stream, note, chord, etc. Pre-v0.2.27 the executor's
// `try: import music21` silently fell through to {} in Pyodide,
// leaving snippets that reference `key.Key(...)` broken with
// NameError. Wheels are vendored locally per the closed-beta
// "no network" guarantee.
await pyodide.loadPackage(["numpy", "pyyaml", "micropip"]);
const micropip = pyodide.pyimport("micropip");
const wheels = [
  "music21-8.3.0-py3-none-any.whl",
  "chardet-<version>-py3-none-any.whl",
  "jsonpickle-<version>-py3-none-any.whl",
  "more_itertools-<version>-py3-none-any.whl",
  "webcolors-<version>-py3-none-any.whl",
];
for (const wheel of wheels) {
  await micropip.install(`emfs:/bundle/pyodide/${wheel}`, { keep_going: true });
}
// Verify import succeeds — fail-fast if a dep is still missing.
pyodide.runPython("import music21; from music21 import key, stream");
```

The `emfs:/` URL prefix tells micropip to read from Pyodide's
in-memory filesystem (where the assets are mounted), bypassing PyPI.

`keep_going: true` lets micropip skip optional sub-deps it can't
resolve (e.g. matplotlib if a transitive dep tree pulls it in
unexpectedly). If the final verification line raises, that's the
real signal — vendor more wheels until it succeeds.

## 8. Don'ts

- Don't pin music21 to a version other than 8.3.0 without a stated
  reason — the snippets in `~/projects/forge-music/blues/` were
  authored against 8.3.0's API (`key.Key`, `roman.RomanNumeral`,
  etc.). Newer music21 may rename or restructure.
- Don't vendor matplotlib or requests just to satisfy a transitive
  warning. They're heavy and unused. Only vendor a dep if a
  `runPython("import music21; ...")` step actually fails for lack
  of it.
- Don't reach for `pip install music21` at plugin runtime. Closed-
  beta has no network.
- Don't skip the size-budget callout in §3 — doubling the zip
  footprint is the kind of decision that needs to be visible to
  the user.
- Don't `--no-verify` or skip pre-commit hooks.
- Don't ship without the §3 in-zip grep confirming the wheels
  actually made it into the artifact. The clean-vault smoke is the
  load-bearing check that the build pipeline + the release-zip
  packager are aligned.

## 9. Protocol comments for driver

Per the v0.2.23-onward convention:

- Whether the per-test Pyodide-boot cost makes this slow enough
  that we should pin music21-bundle tests to a tagged subset (e.g.
  `npm run test:slow` for Pyodide-warm tests, `npm test` default
  excludes them) — vs eating the ~5-10s once per file via the
  singleton pattern, which is the current convention.
- Whether the wheel-list maintenance burden (manifest in three
  places: setup-assets.mjs, build-release-zip.mjs `REQUIRED_FILES`,
  pyodide-host.ts boot loop) suggests a single source-of-truth
  manifest (e.g. `assets/pyodide/manifest.json` listing wheels +
  hashes) consumed by all three. v0.2.27's three-way duplication is
  acceptable for one prompt; a fourth wheel addition would push it
  past worth-extracting.
- Whether `keep_going: true` is the right default or whether
  strictness should match the `try: import music21` silent-skip
  pattern in `executor.py`. The current pattern silently degrades
  the music domain to nothing on import failure — a strict micropip
  install + a strict verification step at boot time would surface
  the gap immediately rather than letting the user discover it via
  NameError on the first compute call. v0.2.27's `keep_going: true`
  + post-install `runPython("import music21; from music21 import
  key, stream")` is a hybrid: silent on transitive sub-dep gaps,
  strict on the modules the snippets actually use. Worth flagging
  as the chosen failure mode.

## 10. One observation for the v1.0 retrospective

This is the third time the "silent-degradation domain bundle"
pattern in `executor.py` has bitten:

- v0.2.x (continuous): moda domain was assumed to work because the
  `try: from forge.moda.types import Particle` succeeded; nobody
  asked whether the moda-side runtime helpers (state serialization,
  particle-loop ticks) were complete.
- v0.2.27 (this prompt): music domain was assumed to work because
  `domains = ["music"]` lit up the dict; nobody asked whether
  Pyodide could actually `import music21`.
- (latent) any future domain (jazz, gtd, ...) added to
  `_DOMAIN_GLOBALS` inherits the same silent-skip-on-ImportError
  pattern. A new domain that depends on a missing wheel ships
  silently broken until a user discovers it via NameError.

**v1.0 audit candidate:** replace the silent-skip pattern with a
fail-loud + a documented domain-availability registry. At engine
boot:

```python
DOMAIN_AVAILABILITY = {}
for domain_name, requires in DOMAIN_REQUIRES.items():
    DOMAIN_AVAILABILITY[domain_name] = all(_can_import(m) for m in requires)
# At vault-load time, if vault declares domains=["music"] but
# DOMAIN_AVAILABILITY["music"] is False, emit a load-time error
# pointing at the missing wheel, not a runtime NameError six call
# levels deep.
```

The current pattern made the v0.2.27 bug invisible across multiple
test suites and 25+ tagged releases. Detection happened only when
a user happened to click Forge on a music snippet for the first
time. Fail-loud at boot is the cheap version of "every
cross-layer contract needs a seam test" applied to the
domain-availability contract.
