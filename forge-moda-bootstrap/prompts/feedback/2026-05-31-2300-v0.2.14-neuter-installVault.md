## 11. One observation

This release moves the pure-core extraction count to six. The
pattern across these six extractions is uniform:

1. A helper function gets identified as testable.
2. Its file imports `obsidian` → fails `node --test`.
3. Pure-core file with a structural adapter type pulled out;
   obsidian-coupled glue file re-exports.
4. Test imports from the pure file.

The cost is one extra file per helper; the benefit is suite-time
catch of API drift and clean test setup. **Worth formalizing as a
project convention before v1.0** — a one-paragraph note in
`cowork-forge-protocol.md` or similar that names the pattern
("new logic that needs unit testing goes in a pure-core file;
obsidian-coupled glue files re-export it") would let future
contributors apply it without having to rediscover the boundary.

The structural-adapter type pattern is also worth naming: each
extraction added its own narrow interface (`CopyAdapter`,
`TomlStubAdapter`). For closed beta this is fine; if the count
grows past ~10, consolidating into a single `MinimalAdapter`
union may be cleaner. Logging as v1.0+ thinking — not a blocker.
