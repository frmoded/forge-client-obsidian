```typescript
const root = existing.getRoot() as unknown as { collapsed?: boolean; expand?: () => void };
if (root?.collapsed && typeof root.expand === 'function') {
  root.expand();
}
this.app.workspace.revealLeaf(existing);
return waitForRealView(existing);
```

## 8. One observation

This is the third successive hotfix on the "render result to user"
hop (v0.2.10 DeferredView placeholder → v0.2.11 reveal-on-existing
→ v0.2.12 expand-collapsed-sidebar). Each iteration uncovered a
different Obsidian-workspace edge case at the boundary between
"compute succeeded" and "user sees something happen". The
underlying signal: that boundary deserves a single named helper
(e.g. `surfaceOutputView`) that encapsulates all the contract
gotchas (placeholder retry, reveal, sidebar expand, future
window-focus, etc.), rather than continuing to bolt new guards
onto `getOutputView`. Worth a v1.0 cleanup pass before more edge
cases surface during the seminar.
