   curl + grep.** The prompt's `grep -A1 '"name": ...' | grep
   '"digest"'` couldn't match because the REST API response's
   `name` and `digest` aren't on adjacent lines. `gh release view
   --json --jq` produces the digest directly. Worth noting as a
   small protocol bullet: "prefer `gh api` / `gh release view`
   over `curl + grep` for GH API field extraction — the JSON
   shape is stable but grep adjacency assumptions are fragile."

   prompt's expected 9)** is a minor structural artifact of the
   `archiver` library used by `build-release-zip.mjs`. The
   directory entry has size 0 and doesn't affect extraction.
   Worth noting in feedback but not actionable.
