## 9. Deviations

- **Bundle test `uvicorn` predicate tightened to `forge.api.server:app`.**
  As written in prompt §5, the substring `uvicorn` would have flagged
  my own v0.2.7 source comment ("With no uvicorn running…") inside
  `main.ts`. The narrower literal is what only the deleted
  `spawnForgeServer` ever emitted, so it's the precise signal. Belt-
  and-suspenders is preserved by the four other symbol/path checks.
- **`Notice` import also dropped from server.ts.** Not explicitly in
  the prompt — discovered while deleting `pingServer` and
  `spawnForgeServer`, the only two `new Notice(...)` callers in the
  file. Leaving an unused import would trip `noUnusedLocals` if the
  project ever turned it on.
- **`bundle.test.ts` as a new file** rather than extending an existing
  test. Per prompt §5 fallback — no existing `main.test.ts`, and the
  assertion is about the build output, not pure-core logic. Keeping
  it separate matches the existing convention.

## 12. One observation

The v0.2.5 → v0.2.8 sequence has been a clean "remove the last
load-bearing residue of the pre-V1 architecture" arc. After v0.2.8,
the only remaining HTTP-only paths in the plugin are explicit user-
invoked commands (`/canonicalize`, `/freeze`, `syncEdgesForActive`)
plus the non-fatal post-/generate dep-sync. That's a clean v1.0
gate: the closed-beta user can run with **zero local processes**,
which was the V1 architecture promise from day one. Worth flagging
in the v1.0 retrospective that closed-beta survived this many
incremental cleanups without breaking — the bundle/test/release
discipline did its job.
